💻 1. El Taller Profesional (Setup)

Sistema: Windows 11 + WSL (Ubuntu) para comandos reales de servidor[cite: 160, 161].

Editor: VS Code conectado de forma remota a Linux[cite: 162].

Control: Git para "puntos de guardado" y GitHub para la nube[cite: 163].

Seguridad: Rama main bloqueada para evitar desastres[cite: 182].

🏗️ 2. El Esqueleto (Semántica HTML5)

No usamos etiquetas por cómo se ven, sino por lo que significan[cite: 165, 166].

  • <main>: El corazón de la página. Solo uno por documento[cite: 66, 168].
  • <strong>: No es solo negrita; es importancia técnica para lectores de pantalla[cite: 91, 171].
  • <blockquote>: Citas que el navegador separa con margen automático[cite: 77, 173, 174].
  • <address>: Información de contacto del autor o local[cite: 71].
Metáfora del Regalo: En el Box Model, el padding es el papel de burbujas que protege el regalo, y el margin es la distancia entre cajas en la estantería[cite: 155].

🎬 3. Dinamismo e Interacción

Video y Audio

Usamos <video> con atributos:

  • controls: Botones para el usuario[cite: 108].
  • muted + autoplay: Obligatorio en navegadores modernos para que arranque solo[cite: 155].
  • poster: Imagen de espera mientras carga[cite: 109].

Formularios de Reserva

Recogen datos mediante <input>, <select> y <button>[cite: 130, 133, 134].

Vinculamos etiquetas con label for="..." para accesibilidad[cite: 131, 155].

🛠️ 4. El Ritual de Git (Comandos)

# Ciclo de trabajo diario:
git status        # ¿Qué he roto/cambiado? [cite: 187]
git add .         # Preparar todo para el viaje [cite: 189]
git commit -m "..." # Etiquetar el paquete de cambios [cite: 191]
git push origin main # Lanzar a la nube de GitHub [cite: 193]

🧹 5. Estándares de Calidad (Clean Code)

Un desarrollador profesional no solo programa, limpia[cite: 175]:

  • Validación: Pasar siempre el validador de la W3C[cite: 48, 155].
  • Indentación: Código ordenado y jerarquizado[cite: 179].
  • Protocolos: Usar mailto: para que el botón de contacto funcione de verdad[cite: 41, 155].
