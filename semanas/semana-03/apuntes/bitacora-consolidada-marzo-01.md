# 📚 Bitácora Completa: De Cero a Desarrolladora Web

## 🤝 1. Experiencia Colaborativa (La Biblioteca)
> **El gran aprendizaje:** No puedes subir tu código (`push`) si la nube tiene cambios que tú no tienes. Primero hay que "traer" lo de los demás con `git pull`.

* **Resolución de Conflictos:** Aprendí a usar la interfaz de VS Code para elegir *"Accept Both Changes"*, integrando mi trabajo con el de compañeros sin borrar nada.

---

## 🏗️ 2. Mentor Urban Toast (Proyecto Individual)
* **Flujo Inicial:** Uso de `git init` y vinculación remota con `git remote add origin`.
* **Rutas Relativas:** Aprendí a no depender de links externos. Ahora uso imágenes locales para que mi web funcione siempre.

---

## 🧹 3. Auditoría y Mantenimiento Profesional
Un desarrollador senior mantiene su "casa" limpia:
* **Adiós a node_modules:** Nunca se suben a GitHub.
* **Limpieza de Basura:** Eliminamos archivos `Zone.Identifier` (basura de Windows).
* **Naming Convention:** Uso de *kebab-case* (ej: `modelo-caja`) para URLs amigables.

---

## 🛡️ 4. Gestión de Seguridad y Pull Requests
* **Alerta de Seguridad:** Gestión de una vulnerabilidad en `rollup` detectada por el robot de GitHub.
* **Bypass de Reglas:** Uso de permisos de administradora para saltar la protección de la rama `main` y autorizar la actualización.
* **Sincronización:** Uso de `git pull` para descargar la corrección al entorno local.

---

## 🏗️ 5. Semántica HTML5 Avanzada
**Metáfora del Regalo:** El *padding* es el papel de burbujas (protección interna) y el *margin* es la distancia entre cajas en la estantería.
